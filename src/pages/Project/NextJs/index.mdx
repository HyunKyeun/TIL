# 들어가며

회사에서 Next JS를 이용해 프론트엔드 개발을 하고 있으면서도  
사용만 할뿐 개념을 깊게 이해 못하여 누군가가 왜 사용하거나 기능을 물어보면 대답이 어렵다.

그래서 짧게 정리하고 들어가려고 한다.

## Next Js?

```
Next Js는 React라이브러리의 프레임워크로,
`SEO(Search Engine Optimization)`를 위한 `SSR(Server-Side Rendering)`이 가능하다 말한다.
기존 React JS의 경우, `Client Side Rendering(CSR)`만이 가능했는데,
이는 웹사이트를 불러올때 빈 html을 불러온 다음 로딩을 통해 해당 페이지의 구성 요소를 다 불러오게 된다.
또한 `SEO(Search Engine Optimization)`에 취약하다
```

라고하는데...

이해한 바로는 CSR의 경우 새로고침시 빈 html이 그려진 다음, 매번 그림,기능을 불러오게 되는점  
또한 `SEO(Search Engine Optimization)`의 경우 구글, 네이버 등 검색엔진 사이트등에서  
미리보기를 통해 보여줄때 load 할수있는 페이지를 미리 불러와 보여주게 되는데,

React 기반으로 만들어진 페이지의 경우 빈 페이지만 보여서 어떤 홈페이지인지 알기 힘들다는 점  
그런데 Next JS의 경우 미리 보여줄 수 있는 겉 페이지가 있어서 노출이 편한 점 이라고 생각한다.

## Next JS의 다른 기능

그 외에도 다른 프레임워크보다 좋은 장점으로

### 페이지 기반 라우팅 시스템

처음부터 이렇게 사용해서 이것이 장점? 이라는 생각도 들었으나,
`/pages` 디렉토리의 폴더 자체가 웹페이지 라우트 주소가 된다.

예를들면 `/pages/user/info.tsx`로, `info.tsx`파일에 작업해둔 페이지가
`http://0.0.0.0/user/info`라는 웹페이지 라우트 주소가 된다.

### Client-Side Navigation

Next에서는 `<Link/>`를 통해 페이지를 이동할수 있다.
기존의 `a`태그의 경우, 페이지를 리로딩 하여 이동하나,  
Link의 경우 해당 페이지 router에 컴포넌트를 불러온다.

짧게 차이를 말해보자면 a 태그의 경우 화면 깜박임을 느낄수있고.. Link는 아니다 라는 느낌.

## 그 외

그 외에도 NextJS 자체 버전도 업그레이드 되고있고, 개발자 측면에서 사용하기 편한 라이브러리도 많이 배포 되고있는 만큼,
기본을 확실히 이해하고 있다면, NextJS로 이것도 가능할까? 싶은 것들을 개발하는데 지장 없지 않을까 싶다.
