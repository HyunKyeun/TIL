# scroll-bar

스크롤바는 `overflow` 속성이 있고, content가 영역을 벗어날때 확장된 영역을 볼 수 있게 해준다.  
당연히 x축으로도 y축으로도 초과할수 있기에 스크롤은 두가지 형태가 존재한다.

사용에 용이함에 따라 x축 스크롤보다 y축으로만 있는것이 편하다.

스크롤을 직접 클릭해서 드래그하는것이 아닌, 마우스의 휠 기능을 이용하기때문에,
x축 스크롤의 경우 그렇게 좋은 스타일이 될수는 없다.
(모바일 버전의 경우는 다르다)

그러나 이 스크롤의 경우 브라우저 내 기본으로 제공하는 스크롤 바의 형태이며,  
커스텀을 하기위해서는 여러 스타일을 수정해야한다.

## overflow-y:auto

y축 오버플로우를 자동으로 생성 auto 대신 scroll로 생성시, 무조건 스크롤바가 생겨있기 때문에 피해야한다.

## --webkit-scrollbar

그 후 --webkit-scrollbar의 속성을 수정함으로써 스크롤바의 스타일을 수정할 수 있다.

- `display:hidden`을 이용하면 스크롤 기능은 있으나, 스크롤이 보이지 않는, 깔끔한 디자인을 만들 수 있다.
  그러나 스크롤이 전혀 안 보이기 때문에 따로 디자인을 수정 하지 않으면, 스크롤 존재 유무를 모를 수도 있다.

- `width:@@` 스크롤바의 두께 조절 가능

-`padding:@@` 스크롤바에 패딩이 생긴다, `margin`도 가능

```javascript
webkit-scrollbar-thumb {
  background: #@@@;
}
```

스크롤 바의 색상 변경이 가능하다.

```javascript
webkit-scrollbar-track {
  background-color: #@@@;
}
```

스크롤 바의 뒷 배경 설정

위 두가지 옵션과 넓이 조절을 통해 얇고 팬시한 스크롤바 생성 가능
(해당 프로젝트의 컬러와 맞는 스크롤바 구현 가능)
